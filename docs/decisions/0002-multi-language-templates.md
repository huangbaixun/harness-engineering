# ADR-0002：多语言模板策略

**状态**：已采纳
**日期**：2026-04-05

---

## 背景

plugin 需要支持不同技术栈的项目。需要决定模板的语言覆盖策略。

## 考虑过的选项

**选项 A：TypeScript 专用**
- 优点：聚焦，模板质量高
- 缺点：排除了大量非 TypeScript 项目

**选项 B：多语言并行模板（TypeScript + Python + Go + 通用）**
- 优点：覆盖面广，每种语言有针对性的最佳实践
- 缺点：维护成本高，需要持续更新

**选项 C：纯通用模板 + 运行时适配**
- 优点：维护简单
- 缺点：缺少语言特有的最佳实践（如 TypeScript 的 strict mode、Python 的 type hints）

## 决策

选择选项 B。提供 generic/ 通用模板作为基础，TypeScript / Python / Go 各有针对性扩展。Skill 在初始化时根据检测到的技术栈自动选择模板。

## 后果（对 Agent 有约束力）

- ❌ 禁止在通用模板中包含语言特有的工具命令
- ✅ 每种语言模板必须包含：CLAUDE.md 模板、Hook 脚本模板、architecture.md 模板
- ✅ generic/ 模板是所有语言模板的超集的概念骨架，不包含语言特有内容
- 下次 review：新增语言模板时，优先评估社区需求量
